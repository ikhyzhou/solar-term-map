24节气气候数据可视化地图

一个基于MapboxGL的Web应用，用于可视化展示中国24节气的降水和气温数据，并结合各地区节气习俗信息进行综合展示。

在线演示：[https://ikhyzhou.github.io/SolarTerm-map/](https://ikhyzhou.github.io/SolarTerm-map/)

项目概述

这是一个大三时期的课程项目，主要实现了：
- 24节气的降水和气温数据地图可视化
- 各地区节气习俗信息的交互式展示
- 节气数据和习俗信息的切换显示功能

核心技术

前端技术栈
- MapboxGL - 地理数据可视化引擎
- HTML5/CSS3 - 页面结构和样式
- JavaScript - 交互逻辑实现
- SVG图标 - 习俗点分类标识

数据处理技术
- ArcGIS - 气象数据清洗和处理
- NetCDF格式处理 - 原始气象数据解析
- GeoJSON转换 - 数据格式转换与优化

主要功能

1. 气候数据可视化
   - 24节气降水量空间分布图
   - 24节气气温分布图
   - 支持两种气候图层的切换

2. 习俗信息展示
   - 各地区节气习俗点的地理分布
   - SVG图标分类标识不同类型习俗
   - 点击坐标点弹出详细信息卡片

3. 交互控制
   - 节气选择器（24节气切换）
   - 气候数据类型切换（降水/气温）
   - 时间轴控制节气时序播放
   - 地图基础交互（缩放、平移）

数据处理流程

项目的数据处理是关键部分，主要包括：

原始NetCDF气象数据
    ↓
ArcGIS数据清洗
    ↓
均值计算与重采样
    ↓
转换为GeoJSON格式
    ↓
MapboxGL可视化渲染

项目亮点

1. 完整的数据处理流程：从原始NC数据到前端可视化的完整链路
2. 多源数据整合：结合气象数据和人文习俗数据
3. 交互式体验：支持多种交互方式探索数据
4. 响应式设计：适配不同屏幕尺寸

学习收获

通过这个项目，掌握了：
- MapboxGL地图开发的基本流程
- 地理数据处理和格式转换
- 气象数据的解析和可视化方法
- 前端地图交互的设计与实现

备注

这是一个学生时期的项目，展示了地理数据可视化的基本能力。项目中使用的数据处理方法和地图交互实现，为后续的WebGIS开发奠定了基础。
